1.68302版本卡件，如果广播和数据帧间隔过近，将导致68302收完广播后来不及处理
  数据帧，从示波器现象来看就是数据帧无返回数据，故此在广播发送完后，延迟一段时间再释放锁
2. 关于几个延时的说明
	一、 iobus_hdlc.c中，hdlc的发收循环中，本来在一个卡件的发收一次完整过程后有一个延时，当时考虑到给卡件发送时发送间隔不能过于紧凑，不过因为卡件此时已经返回数据，所以再次发送应该不会有什么问题，所以该延时可以去掉。
	二、 iobus_common.c中，定时器里每次广播过后，应加入一段延时，该延时之内持有锁，保证68302接收广播后，不要太快又来一次数据帧。
	三、 68302版本，再进行上述修改后，仍存在偶尔超时问题，将广播周期由100ms改为50ms后，问题不再出现
	四、 68302版本，全路扫面第三个字节不返回版码，返回0，与模块指令集不符，715版本无此情况。
